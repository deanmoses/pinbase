# Analytics

## Goals

- Support [public traffic analytics](#public-traffic-analytics)
- Support [product analytics](#product-analytics)
- Support future [public dashboards](PublicDashboardIdeas.md)
- Be privacy-respectful; see [Privacy.md](../../Privacy.md)
- Minimize operational burden, be maintainable by a [small team](../../SmallTeam.md)

### Anti-Goals

- No ad-tech; avoid ad-tech ecosystems
- Analytics do not support [Operational Telemetry](#operational-telemetry)

### Public Traffic Analytics

Purpose:

- pageviews
- uniques
- referral sources
- search-engine traffic
- social traffic
- popular content
- aggregate discovery trends

Characteristics:

- lightweight
- privacy-respectful
- aggregate-focused
- minimal cookies
- no ad-tech integrations

### Product Analytics

Purpose:

- editor workflow analysis / edit funnels
- abandoned edits
- upload funnels
- search success
- feature usage
- search UX
- contributor onboarding
- moderation workflows
- product usability
- discovery metrics

### Operational Telemetry

Operational telemetry is not an analytics concern; they serve different purposes. Retention policies and access controls may differ between systems.

Examples of operational telemetry:

- server logs
- performance metrics
- error tracking
- abuse detection
- security auditing
